MedWatch AI — Replit Agent Brief

1. APP OVERVIEW
App Name: MedWatch AI
Tagline: Real-time patient safety intelligence from fragmented signals to actionable insights.
Core Problem It Solves:
Patient safety signals (adverse drug reactions, hospital complaints, treatment complications) are scattered across social media, WhatsApp forwards, and informal channels in India. No unified system monitors, classifies, or clusters these signals in real time — causing delayed responses to emerging health risks.
Target Users:

Public health officials and pharmacovigilance officers
Hospital quality & safety teams
District health officers (urban: Bengaluru; rural: Raichur)
NGO health workers and field coordinators


2. FULL FEATURE LIST
🔴 Core (Must Have)

Signal Ingestion Dashboard — Accept and display incoming patient safety reports from multiple simulated sources (social media text, WhatsApp-style messages, hospital feedback forms)
AI-Powered NLP Analysis — Send report text to Claude API; extract: drug/symptom entities, risk classification (Critical / High / Medium / Low), signal category (ADR, hospital issue, outbreak signal, treatment complication)
Real-Time Risk Feed — Live scrolling feed of all incoming signals with color-coded risk badges
Geographic Cluster Map — Interactive India map (Leaflet.js) showing signal clusters by district/city with heatmap overlay; filter by Bengaluru (urban) vs Raichur (rural)
Signal Detail View — Full breakdown of each report: original text, NLP-extracted entities, risk level, category, location, timestamp, recommended action
Analytics Dashboard — Charts showing: signals over time, risk distribution pie chart, top drugs flagged, top hospitals mentioned, urban vs rural breakdown
Manual Report Submission — Form for health workers to manually submit a patient safety report
Alert System — Auto-generate alerts when 3+ Critical signals cluster in the same district within 24 hours

🟡 High Value (Should Have)

Trend Detection Panel — Surface emerging patterns: "5 ADR reports mentioning Metformin in Raichur in last 6 hours"
Source Tagging — Tag each signal by source type: Social Media / WhatsApp / Hospital Form / Field Worker
Export Report — Download filtered signals as CSV
Search & Filter — Filter feed by risk level, category, source, date range, location
Signal Timeline — Chronological view of how a cluster evolved over time

🟢 Nice-to-Have

Multilingual Input Simulation — Show Kannada/Hindi text being processed and transliterated
Recommended Actions Panel — AI-generated response recommendations per signal cluster
Dark Mode Toggle

6. DATA & APIS
Database Schema (SQLite)
Table: signals
sqlCREATE TABLE signals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  incident_date DATE,
  source_type TEXT,         -- 'social_media' | 'whatsapp' | 'hospital_form' | 'field_worker'
  location_district TEXT,
  location_type TEXT,       -- 'urban' | 'rural'
  raw_text TEXT,
  reporter_type TEXT,
  drug_name TEXT,
  hospital_name TEXT,
  risk_level TEXT,          -- 'critical' | 'high' | 'medium' | 'low'
  category TEXT,            -- 'adr' | 'hospital_issue' | 'outbreak_signal' | 'treatment_complication'
  extracted_entities TEXT,  -- JSON string
  recommended_action TEXT,
  status TEXT DEFAULT 'new', -- 'new' | 'reviewed' | 'escalated'
  nlp_summary TEXT
);
Table: alerts
sqlCREATE TABLE alerts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  district TEXT,
  signal_count INTEGER,
  time_window_hours INTEGER,
  risk_level TEXT,
  cluster_summary TEXT,
  status TEXT DEFAULT 'active' -- 'active' | 'acknowledged' | 'escalated'
);
API Routes (Express Backend)
GET  /api/signals              — Get all signals (with filters via query params)
GET  /api/signals/:id          — Get single signal
POST /api/signals/analyze      — Submit text → Claude NLP → return analysis + save
PATCH /api/signals/:id/status  — Update signal status
GET  /api/signals/stats/summary — KPI stats (counts, breakdowns)
GET  /api/alerts               — Get all alerts
POST /api/alerts/check         — Run cluster detection, create alerts if threshold met
PATCH /api/alerts/:id/status   — Acknowledge or escalate alert
GET  /api/analytics/timeseries — Signals grouped by hour for last 24h
GET  /api/analytics/drugs      — Top flagged drugs
GET  /api/analytics/hospitals  — Top flagged hospitals
GET  /api/analytics/districts  — District-level breakdown
Claude API Integration
Endpoint: POST https://api.anthropic.com/v1/messages
Model: claude-sonnet-4-20250514
System prompt for NLP analysis:
You are a pharmacovigilance AI assistant for India's patient safety monitoring system. 
Analyze the given patient safety report and respond ONLY with a valid JSON object (no markdown, no explanation) with this exact structure:
{
  "risk_level": "critical|high|medium|low",
  "category": "adr|hospital_issue|outbreak_signal|treatment_complication",
  "extracted_entities": {
    "drugs": ["list of drug names mentioned"],
    "symptoms": ["list of symptoms mentioned"],
    "hospitals": ["list of hospital/clinic names mentioned"],
    "locations": ["list of locations mentioned"]
  },
  "nlp_summary": "2-sentence plain English summary of the safety concern",
  "recommended_action": "1-2 sentence recommended response action for health officials"
}
Risk classification rules:
- critical: life-threatening ADR, death reported, outbreak with 3+ cases
- high: serious ADR requiring hospitalization, systemic hospital failure
- medium: moderate ADR, repeated hospital complaints, 2 linked cases
- low: mild ADR, single complaint, general feedback
